# ADR-006: Proposed Fog–Cloud Learning and Aggregation Framework


## Context

The current system follows a four-tier architecture:

1. Far Edge (Drone)
2. Edge Node
3. Fog Node
4. Cloud Node

The Far Edge performs inference and generates detection outputs.

The Edge Node performs:

* Data reception
* Local storage
* Analytics generation
* Dashboard visualization

No machine learning model training is currently performed at the Edge layer.

The role of the Fog and Cloud layers remains under discussion.

---

## Proposal

The Fog Node should serve as the primary machine learning and intelligence layer within the architecture.

Before information is used for training or higher-level analytics, data generated by multiple Edge Nodes should be combined using an aggregation mechanism.

The Cloud Node should act as a higher-level coordination and model management layer.

---

## Edge-to-Fog Aggregation

Rather than forwarding all Edge Node outputs independently, the Fog Node should first aggregate information from multiple Edge Nodes.

Potential aggregation approaches include:

* Averaging-based aggregation
* Weighted aggregation
* Statistical aggregation
* Temporal aggregation
* Event-based aggregation

Examples:

* Average confidence across Edge Nodes
* Regional sheet-count statistics
* Combined operational metrics
* Time-window summaries

This aggregation layer can reduce communication overhead and provide a cleaner input for downstream learning algorithms.

---

## Proposed Fog Layer Responsibilities

* Receive data from multiple Edge Nodes
* Aggregate Edge Node outputs
* Maintain regional datasets
* Train and update machine learning models
* Generate regional analytics
* Forward model information to the Cloud

---

## Proposed Cloud Layer Responsibilities

* Receive updates from Fog Nodes
* Aggregate regional knowledge
* Maintain global model versions
* Coordinate long-term learning
* Redistribute updated models to Fog Nodes

---

## Potential Research Direction

Investigating aggregation algorithms between the Edge and Fog layers to determine:

* Which aggregation strategy preserves the most useful information
* Communication cost reductions
* Impact on model performance
* Scalability as the number of Edge Nodes increases

Possible techniques include:

* Mean aggregation
* Weighted aggregation
* Hierarchical aggregation
* Federated averaging-inspired approaches
* Custom domain-specific aggregation methods


---

## Expected Outcome

The architecture evolves from a simple data pipeline into a hierarchical intelligence framework where:

* Edge Nodes generate local insights
* Fog Nodes aggregate and learn from regional information
* Cloud Nodes aggregate and coordinate system-wide knowledge
